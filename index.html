Software Engineer Internship

Even Financial

In my previous role as a Software Engineer Intern at Even Financial, I was tasked to solve the problem of not having a consistent way of determining whether a component is in a stable/preferred state. Prior to this, confusion and misinterpretation frequently arose when trying to estimate work and communicate to stakeholders. To apprehend this, I developed a rubric that categorized a given component and determined how independent (“normalized”) it is in comparison to others. This rubric graded a component via several metrics such as whether it used deprecated dependencies, was unnecessarily-complex, or if its business logic was optimally positioned. Then, I utilized the insights from this rubric to dictate how I refactored existing/ created new components. Additionally, I wrote these components with React/Typescript, ensured that they were fully-tested using React Testing Library, and migrated them to a Storybook component library which allows real-time interactivity and inspection of the component in an isolated environment without having to start up a complex dev stack. Ultimately, I was able to help enforce systems that are more sustainable while contributing to how this team can build an ecosystem of components that are suited for scaling up with less risk.
